Starting a Blockchain POC — Minimum Requirements

What does a public agency need to get a proof of concept off the ground? This checklist captures the minimum viable prerequisites identified by the working group.

Technical Prerequisites

  • 1. Wallet software managing cryptographic keys and identity for staff
  • 2. Access to a blockchain infrastructure (not necessarily operated internally)
  • 3. Data retrieval mechanism — API or gateway to read from the ledger
  • 4. Smart contract or application logic for the target use case

Organizational Prerequisites

  • 5. Authority to Operate (ATO) or a defined pathway toward compliance
  • 6. Identified external users who will contribute data to the system
  • 7. Identified external users who will consume data from the system
  • 8. Executive sponsor and program office alignment

See also: GBA Blockchain Maturity Model

Lab & Prototype Ideas

Potential projects to run in a public tech lab setting using test data. Have a question or idea? Open a GitHub issue.

For each prototype proposal, include:

  • • Technology — what platform or protocol
  • • Goal — what question does the prototype answer
  • • Scope of work — what gets built and tested
  • • Organization — who is involved
  • • Point of contact — who to reach out to

Submit proposals as GitHub issues.
