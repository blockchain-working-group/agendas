# Contributing

Welcome! This guide is for anyone who wants to participate—no Git experience required.

## Ways to contribute

- **Propose a discussion topic** — Open an issue using the "Topic Proposal" template
- **Request or volunteer a speaker** — Open an issue using the "Speaker Proposal" template
- **Suggest a project or pilot** — Open an issue using the "Project Proposal" template
- **Add a use case or case study** — Edit the use cases data file or open an issue describing it
- **Share a link or resource** — Open an issue or mention it during the Friday call
- **Fix a typo or update an agenda** — Edit the file directly and submit a pull request

## How to open an issue

1. Go to the [Issues tab](../../issues)
2. Click **New issue**
3. Pick a template (topic, speaker, or project)
4. Fill in the fields and click **Submit new issue**

That's it. Someone from the group will follow up.

## How to edit a file directly on GitHub

1. Navigate to the file you want to change (e.g., `README.md`)
2. Click the pencil icon (top right of the file view)
3. Make your changes
4. Scroll down, add a short description of what you changed
5. Click **Propose changes**
6. On the next screen, click **Create pull request**

A group member will review and merge your change.

## What belongs in the README

The main `README.md` tracks the current year's agendas. When a new year starts, the previous year's agendas get moved into `previous_agendas/`.

To add an upcoming agenda, copy the format of the most recent entry and update the date and topics.
